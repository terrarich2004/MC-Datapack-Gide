## 1. 🏗️ Создание каркаса папок

В отличие от модов, датапаки не требуют компиляции. Это просто набор папок и текстовых файлов. 

Перейдите в папку вашего мира:
`%appdata%\.minecraft\saves\<Имя_Вашего_Мира>\datapacks\`

Создайте там новую папку с названием вашего проекта (например, `MyFirstPack`) и воссоздайте внутри следующую структуру:

```text
📁 MyFirstPack/
│  📄 pack.mcmeta           # Файл-манифест (создадим на шаге 2)
└──📁 data/
   └──📁 minecraft/         # Пространство имен (namespace)
      ├──📁 functions/      # Папка для наших скриптов (.mcfunction)
      └──📁 tags/           # Папка для системных триггеров
         └──📁 functions/
```

> [!TIP]
> Пространство имен `minecraft` используется по умолчанию. В будущем вы сможете создавать свои папки рядом с ней (например, `data/my_cool_map/`), чтобы разделять уровни или механики и избегать конфликтов с другими датапаками.

## 2. 📄 Настройка манифеста `pack.mcmeta`

Файл `pack.mcmeta` говорит игре: *«Привет, я датапак, работающий на такой-то версии!»*.

Создайте в корне папки `MyFirstPack` обычный текстовый файл, переименуйте его в `pack.mcmeta` (убедитесь, что удалили расширение `.txt`) и откройте его через текстовый редактор.

Вставьте следующий актуальный для версий **26.1 / 26.2** JSON-код:

```json
{
  "pack": {
    "description": "Мой первый датапак. Урок 1.",
    "pack_format": 81,
    "supported_formats": {
      "min_inclusive": 80,
      "max_inclusive": 85
    }
  }
}
```

> [!WARNING]
> Начиная с версий 2026 года (26.x), параметры `min_inclusive` и `max_inclusive` в блоке `supported_formats` стали **обязательными**. Если указать только старый `pack_format`, игра выдаст предупреждение о несовместимости.

## 3. 📜 Пишем первую функцию

**Функция** — это текстовый файл с расширением `.mcfunction`. Он работает как идеальная вертикальная цепочка командных блоков.

Перейдите по пути: `data/minecraft/functions/` и создайте файл `hello.mcfunction`.

Напишите внутри следующие команды (каждая с новой строки, **без** знака `/` в начале):

```mcfunction
# Это комментарий, игра его не читает
say Привет, мир! Датапак успешно запущен!
give @a minecraft:cooked_beef 5
effect give @a minecraft:speed 10 1 true
```

### Как проверить работу в игре:
1. Зайдите в ваш мир Minecraft.
2. Введите команду обновления датапаков: `/reload`
3. Запустите функцию вручную: `/function minecraft:hello`

## 4. ⏱️ Автоматизация: теги Load и Tick

Запускать функции вручную неудобно. Чтобы датапак работал сам, привяжем команды к системным событиям игры.

Перейдите по пути: `data/minecraft/tags/functions/`. Создайте здесь два `.json` файла:

### А. Файл `load.json` (Срабатывает ОДИН раз при старте мира или `/reload`)
```json
{
  "values": [
    "minecraft:hello"
  ]
}
```
*Теперь при каждом вводе команды `/reload` игрокам автоматически будет писаться приветствие и выдаваться стейк.*

### Б. Файл `tick.json` (Срабатывает каждый тик — 20 раз в секунду)
Этот файл заменяет **цикличные командные блоки**. Давайте сделаем так, чтобы игра постоянно проверяла какое-то условие.

1. Сначала в папке `data/minecraft/functions/` создайте второй файл: `every_tick.mcfunction`.
2. Напишите в него команду, которая постоянно создает партиклы вокруг всех игроков:
   ```mcfunction
   execute at @a run particle minecraft:happy_villager ~ ~2 ~ 0.5 0.5 0.5 0.01 5
   ```
3. Теперь свяжите этот файл с тиками игры. В папке `tags/functions/` создайте файл `tick.json`:
   ```json
   {
     "values": [
       "minecraft:every_tick"
     ]
   }
   ```

Зайдите в игру, пропишите `/reload` и наслаждайтесь эффектом зеленой пыльцы над головой без единого командного блока в мире!

## 🎯 Практическое задание

Чтобы закрепить материал первого урока, выполните следующее задание:
1. Внутри папки `functions` создайте подпапку с именем `level_1`.
2. Внутри этой подпапки создайте файл `start.mcfunction`.
3. Напишите в него команды для очистки инвентаря игрока (`clear @a`) и выдачи ему стартового набора (например, каменный меч и факелы).
4. Протестируйте запуск этой функции из игры. Подумайте, какой будет правильный путь для её вызова через `/function`? *(Подсказка: папки разделяются косой чертой `/`)*.
